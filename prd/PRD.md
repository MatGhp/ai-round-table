# PRD – Product Requirements Document

**محصول:** AI Round Table
**نسخه:** 1.0 (MVP / PoC)
**وضعیت:** Draft – Ready for Implementation

---

## 1. خلاصه اجرایی (Executive Summary)

**AI Round Table** یک سیستم چندعامله (Multi-Agent AI System) برای تحلیل، نقد، اعتبارسنجی و تصمیم‌گیری روی ایده‌ها است.

این محصول به کاربران کمک می‌کند قبل از سرمایه‌گذاری زمان و هزینه، ایده‌های خود را از زوایای مختلف، متخاصم و ساخت‌یافته بررسی کنند.

### تمرکز محصول:

- تصمیم‌گیری آگاهانه
- حذف سریع ایده‌های ضعیف
- کاهش توهم پیشرفت ناشی از خروجی‌های زیبا اما غلط LLMها

AI Round Table با ابزارهای سنتی هوش مصنوعی از این جهت متفاوت است که اختلاف ساختاریافته، مکانیزم‌های وتوی صریح و شرایط توقف را اجبار می‌کند به‌جای بهینه‌سازی برای ایده‌پردازی یا تولید محتوا.

---

## 2. مسئله (Problem Statement)

کاربران هنگام استفاده از یک LLM واحد برای تحلیل ایده‌ها با مشکلات زیر مواجه‌اند:

- تأیید ناخواسته فرضیات اولیه (Confirmation Bias)
- نبود مخالفت واقعی
- خروجی‌های منسجم ولی غیرواقعی
- رشد بی‌پایان ایده بدون اعتبارسنجی

هیچ ابزار عمومی‌ای وجود ندارد که:

- نقش‌های متضاد را به‌صورت ساختاریافته اعمال کند
- اجازه وتو و توقف ایده را بدهد
- تفکر انتقادی را اجبار کند، نه توصیه

در نتیجه، کاربران اغلب انسجام زبانی را با استدلال اعتبارسنجی‌شده اشتباه می‌گیرند.

## 3. اهداف محصول (Goals)

### اهداف اصلی

- ایجاد تضاد فکری کنترل‌شده
- استخراج و اعتبارسنجی فرضیات
- امکان «نه گفتن» به ایده‌ها
- کمک به تصمیم ادامه / توقف

### اهداف غیرمرتبط (Out of Scope)

- ایده‌پردازی خلاقانه
- تولید محتوا
- جایگزینی Brainstorm انسانی
- چت آزاد یا نقش‌محور صرفاً نمایشی

### Secondary Users

## 5. سناریوی استفاده (User Flow)

1. کاربر ایده را به‌صورت کوتاه وارد می‌کند (5–10 خط)
2. سیستم ایده را به Agentها ارسال می‌کند
3. هر Agent به‌صورت مستقل تحلیل خود را انجام می‌دهد
4. نتایج طبق قوانین سیستم ارزیابی می‌شوند
5. خروجی نهایی شامل:
   - جمع‌بندی
   - ریسک‌ها
   - تصمیم پیشنهادی (Continue / Stop)

---

## 6. معماری مفهومی Agentها

### Agent 1 – Idea Refiner

**هدف:** شفاف‌سازی بدون گسترش

**خروجی:**
- Problem Statement
- Assumptions
- Proposed Solution

### Agent 2 – Reality Checker

**هدف:** افشای فرضیات پنهان

**خروجی:**
- لیست فرضیات
- ادعاهای قابل تست
- نقاط شکست احتمالی

### Agent 3 – Assassin (Veto Agent)

**هدف:** تلاش فعال برای نابودی ایده
**اختیار:** Veto کامل

Assassin Agent به‌صورت صریح برای رد کردن ایده‌ها پاداش می‌گیرد، نه بهبود آن‌ها.

**تمرکز:**
- نبود نیاز واقعی
- راه‌حل‌های ساده‌تر موجود
- Over-Engineering

### Agent 4 – Cost & Complexity Controller

**هدف:** تحلیل Trade-off

**خروجی فقط شامل:**
- هزینه پیاده‌سازی
- هزینه نگهداری
- ریسک عملیاتی
- Cognitive Load

### Agent 5 – Synthesizer

**شرط اجرا:** عدم Veto از Assassin

**خروجی:**
- نسخه نهایی محدودشده
- ریسک‌های باز
- پیشنهاد ادامه یا توقف

## 7. قوانین سخت سیستم (System Rules)

1. هر Agent فقط در محدوده نقش خود اجازه پاسخ دارد
2. حداقل یک Agent باید مخالف ادامه باشد
3. اگر Assassin وتو کند → فرآیند متوقف می‌شود
4. اگر همه Agentها موافق باشند → خروجی Flag می‌شود (Low Trust)
5. اگر خروجی نسبت به ورودی بزرگ‌تر شود → Failure
6. Agentها نباید به‌صورت صریح به یکدیگر ارجاع دهند یا با هم موافقت کنند

---

## 8. الزامات عملکردی (Functional Requirements)

### FR-1
سیستم باید امکان تعریف Agent با موارد زیر را فراهم کند:
- نقش
- LLM اختصاصی
- Prompt اختصاصی

### FR-2
سیستم باید اجرای Agentها را به‌صورت ایزوله انجام دهد

### FR-3
سیستم باید Veto را به‌عنوان وضعیت نهایی ثبت کند

### FR-4
خروجی باید ساخت‌یافته و قابل مقایسه باشد

---

## 9. الزامات غیرعملکردی (Non-Functional Requirements)

- Deterministic تا حد امکان
- Reproducible runs
- Logging کامل تصمیم‌ها
- Latency قابل قبول (< 30s برای MVP)
- Vendor-agnostic (OpenAI, Gemini, Claude, etc.)

---

## 10. معیارهای موفقیت (Success Metrics)

### ❌ موفقیت نیست اگر:

- خروجی فقط طولانی‌تر شود
- همه Agentها موافق باشند
- کاربر حس «خیلی قشنگ شد» بگیرد

### ✅ موفقیت است اگر:

- ایده‌ها زود رد شوند
- فرضیات شفاف شوند
- فرضیات به‌صورت ابطال‌پذیر بیان شوند
- تصمیم توقف سریع گرفته شود
- خروجی کوتاه‌تر از ورودی شود

## 11. MVP Scope

### شامل:

- CLI یا Web ساده
- 3–5 Agent ثابت
- انتخاب LLM برای هر Agent
- خروجی Markdown / JSON
- فایل تنظیمات برای تعریف Agentها (YAML / JSON)

### شامل نمی‌شود:

- UI پیشرفته
- Collaboration
- Memory بلندمدت
- Auto-iteration
